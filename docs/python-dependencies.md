# Dependency Management

> Generated by Claude

This project uses [uv workspaces](https://docs.astral.sh/uv/concepts/workspaces/) to manage a Python monorepo with multiple independent members.

## Project Structure

```
pyproject.toml                              # root (shared deps + workspace config)
├── gen/grpc_buybikeshop_python/pyproject.toml  # package: generated gRPC stubs
├── resources/database/seeder/pyproject.toml    # tool: DB seeder (dev-only)
└── apps/market/pyproject.toml                  # app: e-commerce website
```

## How It Works

### Root `pyproject.toml`

The root file has two jobs:

1. **Declare workspace members** in `[tool.uv.workspace]` — tells uv which directories are part of the monorepo.
2. **Hold truly shared dependencies** in `[project.dependencies]` — only deps that every member needs (e.g. `psycopg2-binary`, `python-dotenv`).

The root should **not** list workspace members (`market`, `seeder`, `grpc_buybikeshop_python`) as dependencies. They are discovered automatically via `[tool.uv.workspace].members`.

### Member `pyproject.toml`

Each member declares its own dependencies. If a member needs another workspace member, it lists it as a dependency and maps it via `[tool.uv.sources]`:

```toml
# apps/market/pyproject.toml
[project]
dependencies = [
    "grpc_buybikeshop_python",  # workspace package
    "starlette>=0.52.1",
    "jinja2>=3.1.6",
]

[tool.uv.sources]
grpc_buybikeshop_python = { workspace = true }
```

### `[tool.uv.sources]` in root

Keep source mappings in the root for any workspace package that is referenced by other members. This lets uv resolve cross-member dependencies during lockfile generation:

```toml
# pyproject.toml (root)
[tool.uv.sources]
grpc-buybikeshop-python = { workspace = true }
market = { workspace = true }
seeder = { workspace = true }
```

## Common Operations

### Install deps for a specific member

```bash
# Install only market and its dependencies
uv sync --package market

# Install only seeder and its dependencies
uv sync --package seeder
```

This installs the member's declared dependencies (including any workspace packages it depends on) without pulling in unrelated members.

### Install everything (all members)

```bash
uv sync --all-packages
```

### Add a dependency to a member

```bash
# Add a package to market
uv add --package market "httpx>=0.27"

# Add a dev dependency to root
uv add --dev "ruff>=0.4"
```

### Run a command in a member's context

```bash
uv run --package market uvicorn market.main:app
uv run --package seeder python -m seeder
```

## Rules

1. **Each dependency belongs to the member that uses it.** Don't put `faker` in root because seeder needs it — put it in `resources/database/seeder/pyproject.toml`.
2. **Root dependencies are for truly universal deps only.** If only one member needs it, it goes in that member's `pyproject.toml`.
3. **Dev tools go in `[dependency-groups]`** in the root (e.g. `grpcio-tools`, `ruff`, `pytest`). These are available via `uv sync --group dev`.
4. **Don't list workspace members as root dependencies.** They are apps/tools, not libraries the root consumes. Cross-member deps are declared in the consuming member's `pyproject.toml`.

## Lockfile

uv generates a single `uv.lock` at the workspace root covering all members. This ensures consistent versions across the monorepo. The lockfile is updated automatically when you run `uv sync` or `uv add`.
